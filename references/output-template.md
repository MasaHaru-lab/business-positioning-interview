# 商业定位文档模板

> 访谈结束后用此模板输出。**核心原则：尽可能使用受访者原话**，而不是 AI 改写过的"商业书面语"。原话才有体温。

---

## 文档结构

```markdown
# 【某某】的商业定位 · v1

> **访谈日期**：YYYY-MM-DD
> **访谈时长**：约 X 小时
> **文档状态**：v1（初稿，建议 7 天后回看，3 个月后用真实读者反馈修订 v2）
> **核心方法**：用对方的原话作为脚手架，不替你重写

## 三、5 个差异化标签（5 秒能记住）

不是"专业、用心、走心"——这是底线，不是差异化。

应该是：
1. **标签 1**：（具体、有画面感、最好带数字或独特经历）
2. **标签 2**：……
3. **标签 3**：……

**辨析：你的差异化是哪一类？**
- [ ] **单一稀缺**：市场上做这件事的人很少
- [ ] **复合差异化**：单独看不稀奇，组合在一起只有你（**大多数人的真实差异化来源**）
- [ ] **视角差异化**：做的是同一件事，但站在独特角度
- [ ] **资源差异化**：能调动别人调动不了的资源

## 七、盲区与风险（坦白告知）

> 这是访谈中你提示但受访者还没完全想透的问题。**不要回避**——回避就是不专业。

| 盲区 | 风险 | 建议下一步 |
|---|---|---|
| 例：定价依据还不清晰 | 容易被砍价或自己心虚 | 找 5 个目标客户聊一次，问他们为类似服务最近一次付了多少钱 |
| 例：差异化还停留在'更用心'层面 | 同行同质化中没有抓手 | 找出 3 个独有的具体故事 / 资源 / 方法，重写差异化标签 |
| 例：资源池和目标客户群不重合 | 冷启动困难 | 设计引流品，先在已有人脉圈做小规模验证 |
| **如果用户设了 boundary**（不愿展开的过去 / 经历） | 可能会缺少关键叙事素材 | 不强求公开，但建议挑 1-2 个可分享片段作为内容母题 |
| **如果用户表现"AI 帮了所以不算我的"心态** | 永远不敢公开自己的 AI 协作产出 | 重新框定：AI 时代独有贡献是 taste / 判断 / 原始素材 / 风格 |
| **如果用户提到"公益 to start with 因为不自信"** | 永远不会拿到"我值多少钱"的市场答案 | 第一天就标价，哪怕极低 |
| **如果用户能量 / 健康有限制** | 高强度产品形态会被身体否决 | 产品体系按"低能量"设计，定期监控能量预算 |
```

## 写作风格守则（写文档时必须遵守）

1. **不用商业术语腔**：避免"赋能"、"打造"、"链接"、"沉淀"、"抓手"、"闭环"——除非受访者自己用了这些词
2. **数字优于形容词**：能写"3 个月"、"5000 块"、"10 个客户"，就不要写"短期内"、"一定金额"、"少量客户"
3. **每个建议都要有依据**：每条结论能追溯到访谈中的某一句话
4. **承认不确定**：v1 是初稿，不要假装它是终极定稿。鼓励用户 7 天后回看、3 个月后修订
5. **温度 > 完美**：用对方的语气，而不是 ChatGPT 的语气
6. **直接引号引用 > 复述改写**：用户原话保留双引号，不要替他"翻译"成更"专业"的话
7. **盲区不能藏着掖着**：用户没想透的问题必须明示，且配建议的下一步——这是文档专业性的核心
